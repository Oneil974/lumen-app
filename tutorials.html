Tutoriels Fluxo

Des workflows simples, utiles, modifiables.

Fluxo assemble des blocs de haut en bas. Chaque bloc produit une sortie, que le bloc suivant peut lire avec {{last}}. Ces exemples sont volontairement basiques : ils servent de socle pour vos propres automatisations.

Variables : {{input}} = entrée du workflow · {{last}} = sortie du bloc précédent · {{model}} et {{ollama_url}} = configuration Lumen.

Workflow 01

Résumer un texte et afficher le résultat

Collez un texte en entrée, demandez à Ollama de produire un résumé clair, puis affichez-le dans une notification Lumen. C'est le workflow de base pour comprendre {{input}} et {{last}}.

5 minutesOllamaNotification
  1. Créez un nouveau workflow et laissez le déclencheur sur Manuel.
  2. Ajoutez un bloc Progression Lumen pour voir que l'exécution commence.
  3. Ajoutez Ollama (génération) et utilisez {{input}} dans le champ Invite.
  4. Terminez par Notification Lumen avec {{last}} comme message.

Entrée de test : un article, une note longue ou un email.

Workflow 02

Créer une note Markdown dans une sandbox

Ce workflow transforme une idée brute en note structurée, puis l'écrit dans un fichier Markdown. Utile pour capturer des idées, préparer un brief ou archiver une réponse IA.

FichiersSandboxMarkdown
  1. Créez ou choisissez une sandbox, par exemple Notes.
  2. Demandez à Ollama de structurer l'entrée en titre, résumé et actions.
  3. Écrivez le résultat dans notes/idee-du-jour.md.
  4. Ajoutez une notification finale pour confirmer la création.

Astuce : remplacez le chemin par un nom dynamique quand votre workflow sera stable.

⋮⋮▣Ollama (génération){{input}}
⋮⋮▤Écrire un fichier{{last}}
⋮⋮◇Notification Lumen
Si vous utilisez ce workflow avec un agent, cochez l'écriture confinée à sa sandbox plutôt que de cibler une sandbox par nom.
Workflow 03

Nettoyer un texte avant de le copier

Un workflow sans IA, rapide et fiable : il nettoie un texte copié depuis le web, remplace les doubles espaces, harmonise la casse et place le résultat dans le presse-papier.

Sans IAPresse-papierTexte
  1. Utilisez {{input}} comme texte source.
  2. Remplacez les caractères ou espaces indésirables.
  3. Appliquez une casse si nécessaire.
  4. Copiez le résultat dans le presse-papier.
⋮⋮TRemplacer le texte{{input}}
⋮⋮AaChanger la casse{{last}}
⋮⋮▣Copier dans le presse-papier{{last}}
Ce type de workflow est idéal en raccourci clavier ou depuis le mode action : il est instantané et ne dépend d'aucun modèle.
Workflow 04

Recevoir un brief quotidien

Planifiez un workflow qui prépare un court brief chaque matin. Il peut être aussi simple qu'un prompt statique, ou devenir plus avancé en lisant des fichiers de notes, une todo list ou des données externes.

CRONRoutineIA
  1. Dans le déclencheur, choisissez Programmé (CRON).
  2. Utilisez par exemple 0 8 * * * pour lancer à 8h chaque jour.
  3. Ajoutez un bloc Texte qui contient votre contexte du matin.
  4. Demandez à Ollama un brief court et affichez-le.

Déclencheur : 0 8 * * * · tous les jours à 8h.

⋮⋮◷Déclencheur programméCRON
⋮⋮TTexte
⋮⋮▣Ollama (génération){{last}}
⋮⋮◇Notification Lumen{{last}}
Le déclencheur n'est pas un bloc dans Fluxo : il se règle dans la barre "Déclencheur" au-dessus du canvas. Il est représenté ici pour rendre le tutoriel lisible.
Workflow 05

Appeler une API puis résumer la réponse

Ce workflow montre la passerelle entre automatisation classique et IA. Il récupère une URL, envoie le résultat au modèle, puis garde une trace dans un fichier.

HTTPAPIRésumé
  1. Ajoutez un bloc Requête HTTP (GET) avec l'URL de votre API.
  2. Envoyez {{last}} dans Ollama avec une consigne de synthèse.
  3. Écrivez le résumé dans un fichier de log.
  4. Affichez une progression pour suivre l'exécution en direct.
⋮⋮↗Requête HTTP (GET)
⋮⋮▣Ollama (génération){{last}}

Workflow 06

Heartbeat : lancer un agent de veille

Ce workflow est fait pour être appelé par le Heartbeat de Lumen. À chaque tick, il lit un contexte partagé, ajoute une tâche à un agent, lance l'agent, puis dépose une trace dans la messagerie du workspace.

HeartbeatAgentMémoireWorkspace
  1. Créez un workflow nommé par exemple Heartbeat veille.
  2. Dans les paramètres Lumen, activez Heartbeat et mettez ce nom de workflow comme cible.
  3. Utilisez une fréquence raisonnable, par exemple 0 */30 * * * *.
  4. Laissez l'agent faire son travail dans sa sandbox, puis suivez le résultat dans la messagerie.

Réglage Heartbeat : cible Heartbeat veille · CRON 0 */30 * * * *.

Le Heartbeat n'est pas un bloc Fluxo : il exécute périodiquement un workflow nommé depuis les paramètres Lumen. La première carte est donc un rappel de configuration, pas un bloc à ajouter au canvas.
Workflow 07

Routeur : classifier une demande et répondre

Le mode Routeur de Lumen appelle le workflow Fluxo default router. Ce tutoriel reprend sa base : afficher une progression, envoyer la demande au modèle avec une consigne de routage, puis retourner une réponse actionnable.

Routeurdefault routerOllama
  1. Ouvrez Fluxo et sélectionnez default router.
  2. Gardez le bloc de progression pour que la conversation affiche l'état du routage.
  3. Adaptez uniquement la consigne système si vous voulez router vers vos propres habitudes.
  4. Testez depuis le mode Routeur de la barre Lumen avec une demande courte.

Entrée routeur : {{input}} = message utilisateur dans la barre Lumen.

Workflow 08

Lire un flux RSS et recevoir les news utiles

Ce workflow récupère un flux RSS public, extrait les titres importants avec Ollama, puis affiche un résumé court dans Lumen. Il est pratique en lancement manuel, en workflow Heartbeat, ou comme sous-workflow appelé par un agent de veille.

RSSHTTPOllamaNotification
  1. Ajoutez une Requête HTTP (GET) vers l'URL du flux RSS.
  2. Envoyez le XML récupéré dans Ollama (génération).
  3. Demandez au modèle de garder seulement les news utiles et les liens.
  4. Affichez le résultat dans une notification ou envoyez-le dans la messagerie.

Entrée de test : laissez vide, le flux RSS vient directement de l'URL.

⋮⋮◌Progression Lumenstep
⋮⋮↗Requête HTTP (GET)
⋮⋮▣Ollama (génération){{last}}
⋮⋮◇Notification Lumen{{last}}
Fluxo n'a pas encore de bloc parseur RSS/XML dédié : ici, le XML est récupéré par HTTP puis filtré par le modèle. Pour un workflow critique, ajoutez plus tard un bloc RSS natif ou un proxy qui transforme le flux en JSON propre.
